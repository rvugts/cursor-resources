# Specification-Driven Development (SDD) Phase Prompts

## Phase 1: Specification
```
Create a comprehensive specification for [feature/system] that includes:
- Functional requirements
- Non-functional requirements
- API design (if applicable)
- Data models
- User stories
- Acceptance criteria
- Technical constraints

Format the specification following the @spec-template.md structure.
```

## Phase 2: Design Review
```
Review the specification for [feature/system] and:
- Identify potential issues or ambiguities
- Suggest improvements
- Validate technical feasibility
- Check for missing requirements
- Ensure alignment with existing architecture

Provide feedback and recommendations.
```

## Phase 3: Architecture Design
```
Design the architecture for [feature/system] based on the specification:
- Component breakdown
- Data flow
- Technology choices
- Integration points
- Scalability considerations
- Security considerations

Create architecture documentation (in /docs/ARCHITECTURE.md) using the @ARCHITECTURE.md template
```

## Phase 4: Implementation Planning
```
Create an implementation plan in (/docs/IMPLEMENTATION_STATUS.md) for [feature/system] that includes:
- Task breakdown
- Dependencies
- Estimated effort
- Risk assessment
- Testing strategy
- Deployment plan

Organize tasks in priority order.
```
