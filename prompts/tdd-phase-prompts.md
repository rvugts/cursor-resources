# Test-Driven Development (TDD) Phase Prompts

## Red Phase: Write Failing Test
```
Following TDD principles, write a failing test for [feature/functionality] that:
- Tests the expected behavior clearly
- Has a descriptive test name
- Follows AAA pattern (Arrange, Act, Assert)
- Is isolated and repeatable
- Will fail for the right reason

The test should be in [test file location] using [testing framework].
```

## Green Phase: Make Test Pass
```
Write the minimal implementation needed to make the failing test pass:
- Focus only on making the test green
- Don't worry about code quality yet
- Keep it simple
- Ensure the test passes

The implementation should be in [file location].
```

## Refactor Phase: Improve Code
```
Refactor the implementation while keeping all tests green:
- Improve code readability
- Remove duplication
- Apply design patterns if appropriate
- Optimize performance if needed
- Ensure all tests still pass

Maintain test coverage and functionality.
```

## TDD Cycle for Feature
```
Implement [feature] using TDD:
1. Write failing tests for each requirement
2. Implement minimal code to pass tests
3. Refactor as needed
4. Repeat for next requirement

Follow the Red-Green-Refactor cycle throughout.
```
