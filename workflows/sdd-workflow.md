# Specification-Driven Development (SDD) Workflow

## Overview
SDD ensures clear requirements and design before implementation.

## Phase 1: Specification
1. Gather requirements
2. Document functional requirements
3. Document non-functional requirements
4. Design API (if applicable)
5. Define data models
6. Write user stories
7. Establish acceptance criteria
8. Review and refine specification

## Phase 2: Design
1. Review specification
2. Design architecture
3. Identify components
4. Plan data flow
5. Choose technologies
6. Design integration points
7. Consider scalability
8. Security review

## Phase 3: Implementation Planning
1. Break down into tasks
2. Identify dependencies
3. Estimate effort
4. Assess risks
5. Plan testing strategy
6. Plan deployment
7. Create timeline

## Phase 4: Implementation
1. Set up project structure
2. Implement core components
3. Write tests (TDD)
4. Integrate components
5. Update documentation
6. Code review
7. Testing and QA

## Phase 5: Deployment
1. Prepare deployment
2. Run final tests
3. Deploy to staging
4. Verify in staging
5. Deploy to production
6. Monitor and verify

## Best Practices
- Don't skip the specification phase
- Review and refine before coding
- Keep specification updated
- Use templates for consistency
- Track implementation status
